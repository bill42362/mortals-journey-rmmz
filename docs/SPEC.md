## 1. 專案目的

將中國知名網路小說《**凡人修仙傳**》凡界篇，改編為一款 **RPGMaker MZ** 製作的單機修仙 RPG。

- **第一階段定位**：可玩 **Demo**，終點為「黃楓谷進階築基完成」（含血色試煉）。
- **製作模式**：以 **AI 主導製作**（劇本、文本、程式、事件），人類僅保留最少參與（主要為地圖繪製與驗收）。此本身也是一次「AI 大規模參與遊戲開發」的實驗。

## 2. 目標受眾與量體預估

| 項目 | 預估 |
| --- | --- |
| 核心受眾 | 《凡人修仙傳》原著讀者 / 修仙題材愛好者 / RPGMaker 獨立遊戲玩家 |
| 體驗時長（Demo） | 約數小時（序章～築基一輪核心循環） |
| 量體定位 | **同人 / 二創性質**的單機作品；非商業大作。量體以「能完整體驗核心精神」為準，而非追求 DAU/規模。 |

> 註：實際公開散布與授權方式（同人非商業 vs 其他）為**待確認商業決策**，見 §6。

## 3. 核心體驗主張（四大精神）

本作的差異化不在數值，而在**還原原著的生存哲學**：

| 精神 | 對玩家的承諾 |
| --- | --- |
| **苟，是一種美德** | 謹慎、避戰、埋伏、留後手會被獎勵；莽撞被懲罰。逃跑與算計永遠合法。 |
| **成長是熬出來的** | 境界提升緩慢扎實，每次突破都是有重量的里程碑。 |
| **小瓶是命脈** | 神秘小綠瓶（加速靈藥生長）是玩家的核心優勢，採藥—煉丹—變強循環圍繞它。 |
| **資源即生命** | 丹藥、符籙、靈石稀缺；準備度往往比數值更決定生死。 |

**二創定位**：比原著仁慈、但敬重原著——玩家可憑根性留住部分原著中殞落的角色（圓滿遺憾），但在 canon 關鍵節點以「軟退場」尊重原著走向。

## 4. 第一階段 Demo 範圍

劇情主線：

| 階段 | 章節 | 境界進程 |
| --- | --- | --- |
| 序章 | 青牛鎮 | 凡人 |
| 第一幕 | 七玄門 | 凡人 → 練氣 |
| 第二幕 | 黃楓谷百藥園 | 練氣・撞瓶頸・立志採藥 |
| 第三幕 | 血色試煉 | 練氣・闖關採藥・遇南宮婉 |
| 壓軸 | 築基 | 量產築基丹・突破築基 |

納入的 **7 項核心系統**：小綠瓶、靈草種植/催化、技能功法、煉丹、回合戰鬥、採藥/闖關、輕量殺人奪寶。
（其餘 9 項系統延後至 Demo 之後。）

## 5. 驗收條件

Demo 視為達成，需滿足：

- [ ] 玩家可完整通關 序章 → 壓軸築基。
- [ ] 核心循環「小綠瓶 → 種植 → 採藥 → 煉丹 → 變強」可運作且玩家能感知其價值。
- [ ] 境界系統呈現「撞瓶頸 → 突破事件」的重量感（非經驗條跳級）。
- [ ] 戰鬥中「逃跑/謹慎」是有效且有時最優的選擇；莽撞會受懲罰。
- [ ] 資源（丹藥/符/靈石）具稀缺感，準備度影響生死。
- [ ] 血色試煉呈現殺人奪寶張力，並正確處理「劇情之壁軟退場」。
- [ ] 全程無阻斷性 bug，可順利存讀檔。
