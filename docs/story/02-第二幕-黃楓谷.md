## 1. 劇情大綱

第一幕末，韓立揣著墨大夫的遺澤與《長春功》上半卷，循殘篇線索奔向**黃楓谷·百藥園**——一座在修仙界頗有名氣、遍植天下靈藥之所。本幕是全 demo 的「**經濟正式運轉 + 系統成熟**」之地：七玄門埋下的種子（靈石、煉丹、種植、境界）在此**全部長成核心循環**，而「**資源即生命**」的重量，也在靈石成為硬通貨的這一刻，第一次真正壓上玩家肩頭。

劇情分五段推進：

**段落一・入谷**——韓立輾轉月餘抵達黃楓谷。滿谷楓紅、靈氣氤氳，比七玄門豐饒何止十倍。初入坊市，他才真正吃了一驚：這裡買賣論的全是**靈石**，凡俗銅板無人肯收。墨大夫臨終那句「出了門，你自會掂出它的分量」，此刻才砸實。他懷中那寥寥幾枚低階靈石，竟成了立足的全部本錢（**靈石經濟・正式啟用**，[TECH §9](../TECH.md)）。

**段落二・立足百藥園**——韓立結識瀟灑仗義、在谷中人脈頗廣的**厲飛雨**；憑一手侍弄靈藥的過人本事（與暗藏的小綠瓶），拜入**百藥園**當差。他分得數畦藥田，**規模化種植**就此展開——小綠瓶的綠液成了稀缺而關鍵的加速資源，怎麼分配、先催哪畦，成了日常的權衡（**種植／小綠瓶系統・規模化**）。坊市之上，他也學會了以靈石**兌換丹藥、素材**，並在**藏經閣**首次見識「**功法習得吃靈石**」——修仙界的一切精進，都要靈石開路（**靈石經濟・全面運轉**）。

**段落三・真煉丹**——七玄門只教得起凡間製藥；黃楓谷的**丹房**，才是真正「點石成丹」的地方。韓立習得**靈丹煉製**（**煉丹系統・深化**：ItemCraftingSys 配方升級，以靈草＋靈石為材煉真靈丹），先前墨大夫口中「天生吃製藥這碗飯」的天賦，在此開花結果。他一面煉丹自用、一面藉丹藥與小綠瓶催生的靈藥苦修，**修為節節攀升**，練氣層數穩步推進。（後山採藥途中亦有**護園驅獸**之戰，此時韓立已有《火球術》傍身，「能戰則戰、該走則走」更游刃有餘。）

**段落四・撞築基之壁**——修為推至**練氣圓滿**，韓立卻撞上一堵比「凡人之壁」更厚的牆：**築基之壁**。光靠苦修與靈藥再難寸進——欲入築基，非**築基丹**不可。他從黃楓谷藏經閣補齊了《長春功》**下半卷**（第一幕懸念的落點），也循墨大夫遺留的**《築基丹殘方》**摸清了門道；卻也得知一個殘酷事實：煉築基丹的關鍵靈藥，**年份極高、性烈難尋**，尋常藥圃種不出，天下也只有一處生得——凶名赫赫的**血色禁地**。

**段落五・立志採藥**——恰逢黃楓谷所屬諸派聯合開啟**血色禁地採藥之行**：進去的人，十不存三，卻是煉築基丹的唯一指望。厲飛雨為他剖析利害。韓立於洞府中默然權衡——那地方確實凶險，可他懷裡有小綠瓶：只要能從禁地帶出**幾株母本**，回來便能催生成批，別人一命換一株，他一株當十株用。**苟得住、算得精、備得足**——這正是他的本錢。韓立立志隨行，收拾行裝，奔赴血色禁地（轉場第三幕）。

> **設計定調（重要）**：本幕基調較第一幕**平緩**——是「大戰之後、大險之前」的**築基前夜**。張力不來自流血，而來自**資源的稀缺與抉擇**（靈石怎麼花、綠液怎麼分、修為與保命如何權衡）與**築基之壁的沉重**。黃楓谷是 demo 中韓立難得相對安穩、能踏實積累的一段；但安穩底下，血色禁地的陰影已然逼近。厲飛雨為**正面配角**（非圓滿遺憾對象、本幕不退場），是韓立在修仙路上第一個真正的朋友。

## 2. 章節目標

### 敘事目標
- 完成「凡俗 → 修仙」的認知翻轉：靈石取代銅板成為命脈，玩家親歷「資源即生命」的真正重量。
- 立住黃楓谷「豐饒、規範、卻也講靈石論實力」的修仙界縮影；以厲飛雨的仗義，給韓立（與玩家）一段難得的暖色。
- 深化四大精神：資源（靈石硬通貨、真煉丹、丹藥自給）、小瓶（規模化催生、綠液分配的日常權衡）、成長（練氣圓滿撞築基之壁的沉重）、苟（不逞禁地之勇、以小瓶之利換取「一株當十株」的算計）。
- 落地第一幕移交的四項需求種子；埋下「血色禁地採藥」動機鉤子，銜接第三幕。

## 7. 實作備註（給 M2 內容生產 / 遊戲端）

> 以下涉及 `data/*.json`、自製外掛與共通事件，屬遊戲端工作（受編輯器鐵則約束）；本檔僅規格化，不在此次動工。**本幕大量事件為第一幕已建立介面的複用；經濟／煉丹介面應寫成可複用 CE／外掛，供第三幕、壓軸呼叫。**

### 7.1 境界系統（`MJ_Realm.js`，EV-11/12）— 撞瓶頸演出，非突破
- **練氣層 = 等級**：本幕韓立於練氣區間內**升級累積**（EV-11），推至**練氣圓滿＝練氣 class 等級上限**（EV-12）。
- **撞築基之壁（EV-12）**：練氣等級到頂後，無法靠升級跨入築基——迫使玩家走「取築基丹」的劇情（第三幕採藥 → 壓軸量產＋突破）。此為 `MJ_Realm` 的**撞瓶頸演出**，**不含 Change Class**（築基突破在壓軸）。
- **鏡像變數**：練氣圓滿時可更新鏡像變數／旗標（如 `V_realm=練氣圓滿`），供 EV-13 之後與第三幕事件判斷。
- **本幕明確定義練氣 class 需求**：練氣需有**等級上限**（層數上限），以承載 EV-12 撞壁。**練氣層數上限具體數值**併 [ROADMAP](../ROADMAP.md)「境界職業參數草案」＋ M4 平衡。
> ⚠️ 待辦：`MJ_Realm.js` 尚未實作；第一幕（凡人→練氣突破）＋本幕（練氣升級／練氣圓滿撞壁）＋壓軸（築基突破）合為其完整需求規格。**建議 `MJ_Realm` 一次設計三境界（凡人／練氣／築基）的等級區間、瓶頸、突破介面**，避免分段返工——這正是「先寫第二幕文件再做外掛」的用意。

### 7.2 功法習得（SkillLearnSystem，EV-08/14）— 靈石花費正式啟用
- **EV-08**：以 **SkillLearnSystem 設「道具花費＝低階靈石」**（[TECH §9.3](../TECH.md)），玩家花靈石參悟一門新功法。**建議習得一門實用/防身功法**（如護體術、遁光/身法類；具體功法待技能表定），教會「靈石換功法」機制。
- **EV-14**：《長春功》**下半卷**——第一幕懸念落點（[第一幕 §7.7](01-第一幕-七玄門.md)）。實作可為「藏經閣靈石換」或「劇情任務取得」。下半卷可表現為：①長春功技能升級／解鎖後續效果，或②習得長春功的進階段。**功法跨境界保留**（[TECH §3](../TECH.md)）。
- ⚠️ 首次讓「功法習得吃靈石」——需在 SkillLearnSystem 參數為對應功法設定靈石道具成本；此設定供全遊戲後續功法復用。

### 7.3 靈石經濟與兌換窗（EV-03/07）— 雙層貨幣正式切換
- **EV-03**：靈石經濟啟用的**敘事開關**（坊市拒收銅板）；建議設一旗標標記「已進入靈石經濟」，供後續商店/兌換分歧。
- **EV-07 靈石兌換窗**：原生 shop 綁 gold（銅板），**不能用於靈石買賣**。依 [TECH §9.3](../TECH.md)，以**共通事件兌換窗**實作（選項＋靈石道具數量條件判斷＋變更持有物），或日後一支小 `MJ_` 外掛統一處理。**本幕即此兌換窗的第一個落點**，應寫成可複用 CE（第三幕黑市、壓軸皆用）。
- **銅板**：本幕起**刻意邊緣化**——坊市不收，僅存量不清零（尊重玩家序章/第一幕累積）；可保留零星凡俗用途或純作「認知落差」演出。

### 7.4 煉丹深化（ItemCraftingSys，EV-09/16）— 真靈丹
- **EV-09**：**真煉丹**教學——ItemCraftingSys 配方**深化**：以靈草＋**靈石**（消耗靈石為「火／引」，呼應 [TECH §9.3](../TECH.md) 煉丹吃靈石）煉真靈丹（療傷靈丹、續氣丹等）。與第一幕「凡間製藥」為**同框架、不同配方層級**（凡藥→靈丹）。
- 韓立**製藥天賦**延續第一幕伏筆（`a2_alchemist_talent`），此處由「凡藥天分」進化為「真煉丹天分」，為壓軸量產築基丹鋪墊。
- **EV-16 備藥**：以真煉丹批量煉製療傷／辟毒／續氣諸丹備禁地之行；復用 EV-09 框架。**築基丹本身本幕不煉**（缺主藥；量產在壓軸）。
- **製符日後同框架複用**（[ROADMAP](../ROADMAP.md) 後續清單）。

### 7.5 種植＋小綠瓶規模化（`MJ_HerbFarming.js`／`MJ_GreenBottle.js`，EV-06/16）
- **複用第一幕介面**（[第一幕 §7.3](01-第一幕-七玄門.md)）：小綠瓶「承月光滋生靈液 → 靈液澆灌加速生長」機制不變；本幕為其**規模化 use case**（多畦藥田、綠液分配）。
- **綠液稀缺依舊**：藥田變多、綠液產量不變（月光節流），凸顯「規模越大、分配越要緊」。GreenBottle 需提供「綠液可用量」查詢＋「指定地塊澆灌」介面，HerbFarming 支援**多地塊並行狀態機**。
- **EV-16 催生母本**：戰略級應用——禁地帶回的**母本**（可堆疊 key item / 特殊靈草）以綠液催生成批。此「以少換多」為本幕主題演出核心，也是第三幕採藥回報的乘數（採得越多母本，回谷催生越多築基丹材料）。
> ⚠️ 待辦：`MJ_GreenBottle.js`／`MJ_HerbFarming.js` 尚未實作；第一幕（首個 use case）＋本幕（規模化＋母本催生）合為需求規格。介面應可續用至壓軸（築基丹靈藥量產）。

### 7.6 回合戰鬥（BattleCore，EV-10）
- **護園驅獸**：可見遇敵、可戰可退；此時韓立**已有《火球術》**（第一幕 EV-21 習得），有輸出手段，戰術較第一幕成熟。

- 本幕戰鬥為**輕量點綴**（非主軸）；正規闖關戰鬥密度在第三幕血色試煉。

### 11.6 畫完後交給 AI 的事

你畫完 M09~M13、設好 tileset 與大致地貌、**關閉編輯器**後，把實際 Map ID 告訴我，我接手：

1. 放入 EV-01~EV-18 **事件薄殼**（對話一律 `\tl{a2_xxx}`；本幕 key 先補進 `Languages.tsv`）。
2. 掛鉤各系統 CE/外掛：靈石兌換（EV-07）、功法習得吃靈石（EV-08/14）、真煉丹（EV-09/16）、規模化種植＋小瓶（EV-06/16）、撞壁演出（EV-12→`MJ_Realm`）、護園戰鬥（EV-10 troop）。

4. 第二幕可玩切片 playtest。

> ⚠️ 依賴項提醒：EV-12 需 `MJ_Realm.js`（撞壁演出）、EV-06/16 需 `MJ_GreenBottle.js`＋`MJ_HerbFarming.js`（規模化）、EV-07/兌換需靈石兌換 CE。這些外掛/CE **多數尚未實作**（見 [ROADMAP M1](../ROADMAP.md)）；本幕事件薄殼可先接 VisuStella 現成系統（SkillLearnSystem/ItemCraftingSys/BattleCore）與 placeholder CE，`MJ_*` 相關事件待外掛就緒再接線。**本幕文件即上述經濟／煉丹／境界介面的成熟需求規格來源（尤為 `MJ_Realm` 三境界完整介面的關鍵拼圖）。**
