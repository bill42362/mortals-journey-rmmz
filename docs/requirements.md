# 凡人修仙傳 RMMZ — 專案需求書

> 本文件為專案的原始需求彙整。依照[文件結構規劃](#8-檔案結構規劃)拆解完成後，此檔將被刪除。

## 1. 技術載體與目錄結構

本遊戲使用 **RPGMaker MZ** 製作。

- 遊戲專案資料夾置於 `game/` 子資料夾。
- 其他周邊檔案與資料夾（git、`README.md`、linter、`docs/` 等）置於專案根目錄。

---

## 2. 專案目標與第一階段範圍

目標為在 RPGMaker MZ 上製作中國網路小說《**凡人修仙傳**》凡界篇。

**第一階段 Demo 版終點**：以「在黃楓谷進階築基完成」作為里程碑，包含血色試煉。

劇情主線進程如下：

| 階段 | 章節 | 地點 | 境界進程 |
| --- | --- | --- | --- |
| 序章 | 青牛鎮 | — | 凡人 |
| 第一幕 | 七玄門 | — | 凡人 → 練氣 |
| 第二幕 | 黃楓谷百藥園 | 百藥園 | 練氣・撞瓶頸・立定進血色禁地採藥的目標 |
| 第三幕 | 血色試煉 | 血色禁地 | 練氣・闖關採藥・遇南宮婉 |
| 壓軸 | 築基 | 黃楓谷 | 回來量產築基丹・突破築基 |

---

## 3. 預計實作的遊戲系統

希望以 RPGMaker 製作下列系統：

- 神秘小綠瓶
- 靈草種植 / 催化
- 技能 / 功法
- 典籍參悟
- 煉器
- 煉丹
- 製符
- 法寶戰鬥（甚至法寶自爆）
- 靈寵培育
- 仙法戰鬥
- 傀儡戰鬥
- 符陣煉製 / 使用
- 屍傀戰鬥夥伴

並且儘量還原原著中**殘酷的殺人奪寶機制**。

---

## 4. AI 製作原則

希望將絕大部分工作交由 AI 製作，最小化人工參與：

- 事件系統編寫
- 劇本 / 腳本 / 文本編寫
- 程式編寫

人工參與部分可能僅剩**地圖繪製**。

---

## 5. 核心精神

希望讓玩家體會到《凡人修仙傳》的核心精神：

| # | 精神 | 說明 |
| --- | --- | --- |
| a | **苟，是一種美德** | 謹慎、避戰、埋伏、留後手都應被機制獎勵；莽撞應被懲罰。逃跑、算計與迴避永遠是合法且有時最優的選項。 |
| b | **成長是熬出來的** | 境界提升緩慢而扎實，每一次突破都是有重量的里程碑，而非經驗條跳級。 |
| c | **小瓶是命脈** | 神秘小綠瓶（加速靈藥生長）是玩家相對於世界的核心優勢，採藥—煉丹—變強的循環圍繞著它運轉。 |
| d | **資源即生命** | 丹藥、符籙、靈石都是稀缺的。準備是否充分，往往比數值高低更決定生死。 |

## 8. 檔案結構規劃

最終目的為拆解本文件後刪除 `requirements.md`，目標結構如下：

| 路徑 | 對象 / 用途 |
| --- | --- |
| `docs/SPEC.md` | 寫給非技術人員、專業 PM 閱讀或維護。描述專案目的、用戶量體預估、驗收條件、商業目標、商業決策等。 |
| `docs/TECH.md` | 技術文件。包含但不限於技術選型、專案架構、專案元件規劃、技術債追蹤等。 |
| `docs/ROADMAP.md` | 遊戲開發的工程階段以及追蹤列表。 |
| `docs/story/` | 遊戲各章節的細部資料。包含但不限於劇情大綱、地圖規劃、地圖事件、對話文本等。 |
| `README.md` | 給技術人員初次接觸專案時快速理解用。包含簡短專案描述、各元件架構、如何在本地一鍵運行等。目前尚未進入開發階段，故幾乎無內容。 |
| `CLAUDE.md` | 給 AI 閱讀的 context。包含需記錄但不適合寫入其他檔案的內容、相關決策點、待決策項目等。 |
| `game/` | 暫時留空，目前仍在文件製作階段。 |
